# changelogfa.st — Release Notes Action

> **Release notes that write themselves.**

[**changelogfa.st**](https://changelogfa.st) turns your merged pull requests and commits into clean, human-readable release notes — with a strict no-hype, noise-filtered editorial style. This GitHub Action runs it in your CI: when you publish a release, it generates a polished note and **enriches your GitHub Release in place** (or opens a `CHANGELOG.md` pull request — your choice).

- 🌐 **What is changelogfa.st?** An automated release-note writer for software teams. → [changelogfa.st](https://changelogfa.st)
- 👀 **See it live:** [example releases generated by this Action »](https://github.com/changelogfast/changelog-fast-action/releases)

## Setup (one click, no API key)

1. **Install the [changelogfa.st GitHub App](https://github.com/apps/changelogfa-st/installations/new)** on your repo. That's the only setup — it provides the auth, the write access used to enrich the release / open the PR, and your free monthly quota.
2. Add the workflow below.

```yaml
# .github/workflows/release-notes.yml
name: Release Notes
on:
  release:
    types: [published]

permissions:
  id-token: write        # required — keyless auth via GitHub OIDC

jobs:
  changelog:
    runs-on: ubuntu-latest
    steps:
      - uses: changelogfast/changelog-fast-action@v1
        with:
          output: release   # default: enrich the GitHub Release in place
          style: technical  # or: marketing
```

That's it. On each release, the note is written straight into your GitHub Release. **No API key, and no `contents`/`pull-requests` permissions in your workflow** — changelogfa.st does the writing server-side; the workflow only proves its identity via OIDC (`id-token: write`).

## Inputs

| Input | Default | Description |
|-------|---------|-------------|
| `output` | `release` | What to produce: `release` (enrich the GitHub Release), `pr` (open a `CHANGELOG.md` PR), or `both`. |
| `style` | `technical` | Release note style (`technical`, `marketing`, …). |
| `api-url` | `https://changelogfa.st` | API base URL. Override only for self-hosting/testing. |

## Outputs

| Output | Description |
|--------|-------------|
| `release-url` | URL of the enriched GitHub Release (when `output` includes `release`). |
| `pr-url` | URL of the opened `CHANGELOG.md` pull request (when `output` includes `pr`). |

## How it works

The Action requests a short-lived **GitHub OIDC token** (no stored secret) that proves which repo it runs in. changelogfa.st verifies it, confirms the App is installed on that repo, generates the note from your release, and writes it back — enriching the Release body and/or opening a `CHANGELOG.md` PR. Re-runs are idempotent (the Release section is updated in place; a PR for the same version is reused).
